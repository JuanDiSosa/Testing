# Testing
Proyectos realizados durante el centificado de "Introduction to Software Testing"

Este curso está dirigido principalmente a aquellos estudiantes interesados en cualquiera de los siguientes roles: ingeniero de software, ingeniero de software en pruebas, ingeniero de automatización de pruebas, ingeniero de DevOps, desarrollador de software, programador, entusiasta de la informática. Esperamos que comprenda el lenguaje de programación Java (o cualquier lenguaje orientado a objetos similar y la capacidad de captar la sintaxis de Java rápidamente) y algún conocimiento del ciclo de vida del desarrollo de software.

Después de completar este curso, podrá…
- Describe la diferencia entre verificación y validación.
- Explique el objetivo de las pruebas.
- Utilizar la terminología de prueba adecuada en la comunicación; específicamente: dispositivo de prueba, caso de prueba lógico, caso de prueba concreto, script de prueba, oráculo de prueba y falla.
- Describir las motivaciones de las pruebas de caja blanca y negra.
- Comparar y contrastar las técnicas de desarrollo test-first y test-last.
- Mida la adecuación de la prueba utilizando declaraciones y cobertura de sucursales.
- Razón sobre las causas y aceptabilidad y mala cobertura
- Evaluar la eficacia de búsqueda de fallos de un conjunto de pruebas funcionales mediante pruebas de mutación.
- Criticar las pruebas de caja negra y caja blanca, describiendo los beneficios y el uso de cada una dentro del mayor esfuerzo de desarrollo.
- Distinguir entre los oráculos de prueba de valor esperado (verdadero), heurístico, de consistencia (como se usa en la regresión A / B) y de probabilidad y seleccionar el que mejor se adapte al objetivo de prueba.
- Cree casos de prueba de integración y unidad para detectar defectos dentro del código y automatizar estas pruebas usando JUnit. Para lograr esto, los estudiantes emplearán pruebas dobles para respaldar sus pruebas, incluidos talones (para verificación estatal) y simulacros (para verificación de comportamiento) (https://martinfowler.com/articles/mocksArentStubs.html)
